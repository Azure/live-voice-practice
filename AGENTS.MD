# Live Voice Practice Platform

This repository contains a real-time AI-powered voice training platform for call-center and customer-service practice. It combines a React client, a Flask API/WebSocket backend, Azure Voice Live, Azure AI services, and Bicep infrastructure for public or network-isolated Azure deployments.

## Key Features Implemented

### Real-Time Voice Practice

- Browser-based practice sessions where trainees select a scenario and speak with an AI-powered customer.
- Backend WebSocket proxy at `/ws/voice` that bridges browser events with Azure Voice Live.
- Real-time audio playback, transcript capture, and voice/avatar session handling in the frontend.
- WebRTC support for avatar media connection setup and diagnostics.

### Scenario-Based Training

- Server-side scenario loading through `GET /api/scenarios`.
- Scenario definitions seeded from `samples/scenarios/`.
- Scenario prompts and runtime agent behavior managed in backend services.
- Graph-based scenario generation support using canned sample data in `data/`.

### Conversation Analysis and Feedback

- Post-session analysis through `POST /api/analyze`.
- Structured scoring and feedback generated by Azure OpenAI through Azure AI Foundry / Azure AI Services endpoints.
- Pronunciation assessment through Azure Speech.
- Reusable evaluation rubrics under `samples/rubrics/`.
- Sample transcripts and audio examples under `samples/transcripts/`.

### Conversation History and Trainer Views

- Optional Microsoft Entra ID sign-in.
- User-aware practice history through conversation APIs.
- Trainer-oriented "All Practices" views when authenticated user roles allow it.

### Support Materials Search

- Optional Azure AI Search integration for support/training materials.
- Sample support documents under `samples/materials/`.
- Backend search service can enrich analysis with indexed supporting materials when configured.

### Azure Deployment and Operations

- Azure Developer CLI (`azd`) workflow wired through root `azure.yaml`.
- Bicep infrastructure under `infra/`.
- Basic public deployment mode for development and demos.
- Network-isolated deployment mode with private endpoints, private DNS, jumpbox/Bastion workflow, Azure Firewall, and optional Application Gateway WAF v2 public ingress.
- Post-provision and post-deploy scripts for data seeding, configuration, and smoke validation.

## Technology Stack

### Backend

- **Python**: Backend application language.
- **Flask 3.1.3**: HTTP API and static frontend hosting.
- **Flask-Sock 0.7.0**: WebSocket endpoint support.
- **Azure Voice Live SDK (`azure-ai-voicelive`)**: Real-time voice session integration.
- **Azure AI Projects (`azure-ai-projects`)**: Azure AI Foundry project integration.
- **OpenAI Python SDK 2.13.0**: Azure OpenAI chat completion and embedding workflows.
- **Azure Speech SDK (`azure-cognitiveservices-speech`)**: Pronunciation assessment and speech-related capabilities.
- **Azure Cosmos DB SDK (`azure-cosmos`)**: Scenario and conversation persistence.
- **Azure AI Search SDK (`azure-search-documents`)**: Support materials retrieval.
- **Azure App Configuration SDK**: Runtime configuration integration.
- **Azure Identity**: Managed identity and local Azure credential support.
- **pytest, flake8, pylint, ruff, black, mypy**: Backend testing, linting, formatting, and type-checking tools.

### Frontend

- **React 19**: UI framework.
- **TypeScript 5.9**: Type-safe frontend development.
- **Vite 7**: Frontend build and development server.
- **Fluent UI React v8 and v9**: UI components and design tokens.
- **ESLint 9 and Prettier 3**: Frontend linting and formatting.
- **Custom React hooks**: Audio recording/playback, real-time WebSocket handling, WebRTC setup, scenarios, authentication, and conversation history.

### Azure Services and Infrastructure

- **Azure Container Apps**: Hosts the combined app container.
- **Azure Container Registry**: Container image storage and, in isolated mode, VNet-integrated ACR Tasks builds.
- **Azure AI Foundry / Azure AI Services**: Model and project backing services for analysis and AI workflows.
- **Azure Voice Live**: Real-time AI voice conversation capability.
- **Azure Speech**: Speech and pronunciation assessment.
- **Azure Cosmos DB**: Scenario and conversation data store.
- **Azure AI Search**: Support materials indexing and retrieval.
- **Azure Key Vault**: Secret and certificate storage.
- **Azure App Configuration**: Runtime configuration publishing.
- **Azure Storage**: App and data storage needs.
- **Log Analytics and Application Insights**: Observability.
- **Virtual Network, private endpoints, private DNS, Azure Firewall, Bastion, and optional Application Gateway WAF v2**: Network-isolated deployment topology.
- **Bicep**: Infrastructure as Code.
- **Azure Developer CLI (`azd`)**: Provisioning and deployment orchestration.
- **Docker**: Application containerization.

## Common Development Commands

### Backend

```bash
cd backend
python src/app.py
pytest
black . --config pyproject.toml
flake8 . --config=.flake8
```

### Frontend

```bash
cd frontend
npm run dev
npm run format
npm run lint
npm run build
```

### Azure

```bash
azd env select live-voice
azd deploy
```

Use `azd provision` when infrastructure changes require it. Use `azd up` for the full provision-and-deploy flow.
