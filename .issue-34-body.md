# PR #33 fix is non-functional on Windows: `$env:GIT_CONFIG_VALUE_0 = ''` deletes the variable, breaking the clone

## Summary

`Invoke-GitCloneWithTimeout` in `install.ps1` (v1.1.3 retag `68c2172`, the "fix #33" retag) still fails the cold-start NI clone with the new error:

```
error: missing config value GIT_CONFIG_VALUE_0
At line:14 char:13
+             & git clone -b $t --depth 1 --no-tags $u $d 2>&1
```

The CSE returns exit code `1` and the deployment fails on the jumpbox VM extension. The PR #33 fix attempted to disable Git Credential Manager via the `GIT_CONFIG_COUNT` / `GIT_CONFIG_KEY_n` / `GIT_CONFIG_VALUE_n` protocol, but on **Windows PowerShell the `$env:VAR = ''` syntax removes the environment variable instead of setting an empty value**. That leaves Git with `COUNT=1`, `KEY_0=credential.helper`, but no `VALUE_0` at all, which Git treats as a hard configuration error and aborts before any network I/O.

## Reproduction

1. Provision the AILZ pattern on a Windows jumpbox in a network-isolated subnet (i.e. exercise the CSE path that runs `install.ps1`).
2. CSE log shows:
   ```
   error: missing config value GIT_CONFIG_VALUE_0
       + & git clone -b $t --depth 1 --no-tags $u $d 2>&1
   ```
3. Deployment fails with `VMExtensionProvisioningError` exit code `1`.

Validated end-to-end: 31 of 33 resources provisioned cleanly with retag `68c2172` (Foundry, model deployments, all PEs, container app), only the jumpbox CSE fails — and it fails immediately at the very first clone attempt (the AILZ repo itself), not at any of the extra repos.

## Root cause

In `infra/install.ps1` (https://github.com/Azure/bicep-ptn-aiml-landing-zone/blob/v1.1.3/infra/install.ps1) inside the `Start-Job` script block introduced by PR #33:

```powershell
$env:GIT_CONFIG_COUNT     = '1'
$env:GIT_CONFIG_KEY_0     = 'credential.helper'
$env:GIT_CONFIG_VALUE_0   = ''       # <-- on Windows, this UNSETS the variable
```

On Windows PowerShell, the `Environment:` provider treats assignment of an empty string as a delete:

```powershell
$env:FOO = 'bar'
$env:FOO = ''
[Environment]::GetEnvironmentVariable('FOO','Process')   # -> $null
```

So the child `git` process sees:
- `GIT_CONFIG_COUNT=1` (set)
- `GIT_CONFIG_KEY_0=credential.helper` (set)
- `GIT_CONFIG_VALUE_0` (**not present**)

Git's parser for the `GIT_CONFIG_*` protocol requires `VALUE_n` to exist (even as empty); when it does not, it aborts with `error: missing config value GIT_CONFIG_VALUE_<n>` and `LASTEXITCODE=128`. The error happens before any TLS handshake, so this fails on every clone, NI or not, on Windows.

## Proposed fix

The simplest, idiom-correct fix is to drop the `GIT_CONFIG_*` env-var protocol entirely and pass `credential.helper=` as a `-c` flag on the `git` invocation. `git -c key=` semantically sets an empty value for the given key for that one invocation, and is portable:

```powershell
& git -c credential.helper= clone -b $t --depth 1 --no-tags $u $d 2>&1
```

If the env-var protocol is preferred for any reason, the empty-value assignment must use the .NET API to actually set an empty string (not delete) on Windows:

```powershell
[Environment]::SetEnvironmentVariable('GIT_CONFIG_COUNT',   '1',                'Process')
[Environment]::SetEnvironmentVariable('GIT_CONFIG_KEY_0',   'credential.helper','Process')
[Environment]::SetEnvironmentVariable('GIT_CONFIG_VALUE_0', '',                 'Process')
```

`SetEnvironmentVariable(name, '', 'Process')` keeps the variable present with an empty value; only `SetEnvironmentVariable(name, $null, 'Process')` deletes it. The `$env:` provider does not distinguish between those two, so it cannot be used here.

I would recommend the `-c credential.helper=` flag — it removes three env vars, removes a Windows/PowerShell footgun, and is the documented Git way to override config for a single command.

## Patch

```diff
@@ Invoke-GitCloneWithTimeout
-            $env:GIT_TERMINAL_PROMPT = '0'
-            $env:GCM_INTERACTIVE     = 'Never'
-            # Disable Git Credential Manager for this public-clone path.
-            $env:GIT_CONFIG_COUNT     = '1'
-            $env:GIT_CONFIG_KEY_0     = 'credential.helper'
-            $env:GIT_CONFIG_VALUE_0   = ''
-            $env:GIT_HTTP_LOW_SPEED_LIMIT = "$lim"
-            $env:GIT_HTTP_LOW_SPEED_TIME  = "$tsec"
-            & git clone -b $t --depth 1 --no-tags $u $d 2>&1
+            $env:GIT_TERMINAL_PROMPT      = '0'
+            $env:GCM_INTERACTIVE          = 'Never'
+            $env:GIT_HTTP_LOW_SPEED_LIMIT = "$lim"
+            $env:GIT_HTTP_LOW_SPEED_TIME  = "$tsec"
+            # Disable Git Credential Manager for this public-clone path
+            # via -c flag (avoids the Windows `$env:VAR = ''` delete trap).
+            & git -c credential.helper= clone -b $t --depth 1 --no-tags $u $d 2>&1
             "__GIT_EXIT__:$LASTEXITCODE"
```
